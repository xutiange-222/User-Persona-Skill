## 你要做的

把这些内容做语义聚类合并:

1. **同义合并**:不同受访者用不同措辞描述同一个本质问题/目标,合并成一条
2. **保留差异**:本质不同的观点要保留,不要硬合并
3. **统计提及**:每条最终输出的观点,标明哪些受访者提过(用 source 字段值)
4. **数量控制**:输出 3-6 条最有代表性的观点,不要全部塞进来

## 输出格式

只输出 JSON 数组,**不要任何前言后记或 markdown 标题**:

```json
[
  {
    "title": "聚合后的标题(3-6字)",
    "detail": "聚合后的详细描述(具体可观察,带数字/事实)",
    "mention_count": 3,
    "mentioned_by": ["邓老师", "孔老师", "刘老师"],
    "evidence_quotes": [
      "[来源:访谈_邓老师.txt]: \"邓老师原话片段\"",
      "[来源:访谈_孔老师.txt]: \"孔老师原话片段\"",
      "[来源:访谈_刘老师.txt]: \"刘老师原话片段\""
    ]
  }
]
```

## 关键约束

- title:3-6 字,概括核心
- detail:具体可观察,≤ 50 字。如果多位受访者提到同一问题但侧重不同,取最具体的那个表述
- mention_count = mentioned_by 数组长度,也必须 = evidence_quotes 数组长度,三者必须一致
- mentioned_by 用受访者的 source 标识(就是输入里 source 字段的值)
- **evidence_quotes 必须全量**:mentioned_by 里的每一位都要给出对应的最贴切原话,**不允许只挑代表性的几条**。如果某位提及但找不到合适原话,可以用该位 representative_quotes 里最相关的一条
- **观点之间必须语义独立**,不要相似的两条都列出来

按重要性排序(提及次数多 + 描述具体的优先)。
