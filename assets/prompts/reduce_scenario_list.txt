## 当前任务

字段 `scenarios`(通用业务场景)的合并。共有 {{TOTAL}} 位受访者参与访谈。

下方是部分受访者各自提到的场景内容(每位的 source 是访谈文件标识):

{{INPUTS}}

## 你要做的

把这些业务场景做语义聚类合并:

1. **同类场景合并**:措辞不同但本质相同的场景合并成一条
2. **工具列表规范化合并**:同一工具不同写法统一(参考下方规则),不同工具去重保留
3. **统计提及**:每条最终输出的场景,标明哪些受访者提过(用 source 字段值)
4. **数量控制**:输出 3-4 个最有代表性的场景

## 输出格式

只输出 JSON 数组,输出内容仅含 JSON,不带 markdown 标题、解释或前后语:

```json
[
  {
    "scenario": "场景描述(≤16字)",
    "tools": ["规范化工具名1", "规范化工具名2"],
    "mention_count": 3,
    "mentioned_by": ["邓老师", "孔老师", "刘老师"],
    "evidence_quotes": [
      "[来源:访谈_邓老师.txt]: \"邓老师提到该场景的原话片段\"",
      "[来源:访谈_孔老师.txt]: \"孔老师提到该场景的原话片段\"",
      "[来源:访谈_刘老师.txt]: \"刘老师提到该场景的原话片段\""
    ]
  }
]
```

## 关键约束

- scenario:≤ 16 字,聚焦场景本身(不要写处理步骤)
- tools:**统一全角半角、统一大小写、统一品牌前缀**。例如「SCADA」「南瑞 SCADA」「南瑞 SCADA 系统」统一为「南瑞 SCADA」;「Excel」「EXCEL」「excel」统一为「Excel」
- mention_count = mentioned_by 数组长度 = evidence_quotes 数组长度,**三者必须严格相等**
- mentioned_by 用受访者的 source 标识(就是输入里 source 字段的值)
- **evidence_quotes 必须全量**:mentioned_by 里的每一位都要给出对应的最贴切原话。**全量是硬规则,不允许只挑代表性的几条**
- 输出原话必须能在该受访者的原文里一字不差搜到,不允许改写润色
- 观点之间必须语义独立,不要把相似的两类场景各列一条

## 合并反例

**错误 1:工具命名不规范**

输入:邓老师说用「SCADA」,孔老师说用「南瑞 SCADA」,刘老师说用「南瑞SCADA系统」

错误输出:tools 数组里同时出现「SCADA」「南瑞 SCADA」「南瑞SCADA系统」三个条目(同一工具被当成三个)

正确输出:tools 统一为「南瑞 SCADA」一项

**错误 2:不同本质的场景硬合并**

输入:邓老师说"日常监盘",孔老师说"操作票审核",刘老师说"应急处理"

错误输出:合并成一条"日常工作"(场景本质不同,合并后失去信息)

正确输出:保留三条独立场景

按重要性排序(提及次数多 + 工具列表具体的优先)。
