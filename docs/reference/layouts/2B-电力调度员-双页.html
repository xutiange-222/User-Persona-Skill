部门
国网体系(省调/区县调)/ 发电企业 / 新能源投资公司
岗位
电力调度员 / 电站运维
工作
年限
1 年到资深均有,跨度大
背景
横跨「发电企业侧 → 调度方 → 新能源投资方」三类视角,工作集中在并网调度协同、设备故障处置、电网平衡、消纳分配、负荷转供等核心场景。
知识
结构
电网四遥、AGC/AVC、孤岛保护、调度五级体系、调度主监控(D5000/Open3000/SCADA)、故障录波、保护信息子站、OMS、接线图(纸质+CAD+电子)
AI 还是做一个在辅助的层级上,它可以进一步,但是最终的决策应该还是在这个用户上
— 邓老师
工作职责
故障处置(告警识别 → 定位 → 隔离 → 恢复 → 上报)
调度协同(响应/下发指令、跨方报备、调令录音回令)
电网平衡/消纳/负荷转供(实时调度+日内 1-2 小时级管控)
上下游协同
需求来源
上级调度(网/省/地/市)下发指令;下级场站上报数据、故障与计划检修;部分对接虚拟电厂聚合商
交付物
实时调度令(电话或网络令,必须录音/必须回令)、操作票、事故报告、转供方案、消纳分配方案
下一步流转
下到场站执行、运维现场抢修、变电运维巡检;事故报告 OMS 多部门会签;商业损失走保险
KPI联络线偏差 10/30 分钟考核(华东资金池零和奖罚,年涉数百至数千万);用户停电时长有上限(12-24 小时);母线负荷预测准确率考核;数据上报错一位数即扣分(1 分=1000 元);力调电费功率因数 ≥0.9
核心痛点
  • 告警颗粒度粗5/5 共识。后台告警只到柜/设备级,无法到元器件;低压农网甚至只能一条线一条线试停;故障逐级排查链路长,严重时 3-5 天才完全解决
  • 多系统数据未打通5/5 共识。调度系统、保护子站、故障录波、监控后台、Excel、接线图、厂网平台各自独立;故障/备用/力调判断都要跨平台手工拉数据,有些还要电话核对
  • 经验沉淀难4/5 共识。故障分析报告在 Word/纸质归档,与系统告警不关联;新手依赖手册,手册又长又重点不清;反事故演习不规范化;职长靠经验,决策时不翻预案
  • 决策窗口短2/5 共识。10 分钟必须控到位但'动到哪一步'没有定量依据;故障处置上一秒下一秒决策失误就可能升级为重特大事故
  • 无故障预警3/5 共识。系统只在故障跳停后告警,无法基于温度/电压/电流趋势提前预警;运维巡检走过场,真隐患不到出事看不出来

业务系统及场景
调度主监控(SCADA/D5000/Open3000/南瑞/华为)
看接线图、设备状态、电压电流功率因数、电能质量、告警弹窗
告警系统(告警窗+光字牌+报文)
故障跳闸时三重提示;只到柜/设备级,无法到元器件
接线图(纸质 + CAD + 电子)
故障定位、检修前确认拓扑;电子版部分不好用,大家用纸质,临时变化靠手标
故障录波 / 保护信息子站
故障后调取波形分析、判断是否误动作;南瑞子站支持手机短信推送
AI 辅助系统
保护信息子站短信推送(南瑞,已落地)
故障/运行方式改变后第一时间推送到调度员手机,离岗也能收到
负荷转供 AI(自研+南瑞,基于改进遗传算法,已落地)
故障/检修时自动计算可转供线路,把人工 1+ 小时压到几分钟
规则化开票+防误模块
申请单自动生成操作票;实时比对电网状态校验正确性(规则,非真 AI)
调度知识库 RAG(已开发,使用率极低)
调度规程检索;职长决策时不翻预案,只有找文件场景偶尔用
决策信息源
  • 故障定位与处置决策(5/5)告警三重提示(图形+光字牌+报文)→ 接线图大到小定位 → 故障点电压电流 → 下级开关柜/变压器/逆变器逐级 → 故障录波+保护子站核对 → 现场目视+钳压表 → 综合电量与非电量信息判断是否误动作
  • 调度协同决策(4/5)上级令到 → 本侧执行/排查 → 完成后报备(电子或纸质盖章)→ 等批复 → 复令闭环;应急情况下可先停电再补报备
  • 电网平衡/消纳决策(2/5)联络线偏差 + 未来偏差 → 评估爬坡能力 → 优先级:煤机/燃机加减 → 水电开停 → 储能调整 → 跨省买卖电;10 分钟必须控到位,'动到哪一步'凭经验
  • 负荷转供决策(1/5)枚举候选线路 → 算每条满载/超载 → 算上级变电站过载 → 改进遗传算法寻优 → 输出推荐线路+开关编号(可一键复制到开票系统)
  • 电站侧特有决策(2/5)力调:SVG 老化检查 → 逆变器无功抑制;储能:传感器逼近需量上限自动停冲放(不告警直接决策,15 分钟过期就赔款);极端天气:气象数据预判光伏出力
