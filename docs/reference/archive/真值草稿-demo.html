v6.3 箭头方案 — 从根上重新设计:不用 SVG overlay,完全 CSS Grid 内嵌

产品需求(你说的):表达"同一用户角色"的从左到右、从上到下顺序。 不跨角色折线(SVG overlay 必飘,放弃)。

3 种箭头机制,都"长在 grid cell 里"跟着 grid 走,永远不飘:
① 相邻 task→task:cell 加 .has-right-arrow,CSS ::after 在 cell 右边缘画 → 短箭头
② 跨空 cell 的长拉:空 cell 直接换成 <div class="l1-arrow-line">(可加 .span-2 / .span-3 占多列),内部 flex 撑满画长横线 + 三角
③ 同 cell 内多 task:CSS .l1-task + .l1-task 自动在第二个 task 上方画 ↓ 短箭头

真实场景已演示(去仔细数 ↓):
· 研究员行 6 个相邻 → + 1 个 col7→col9 跨空 col8 长拉 + col2 范围对齐↓背景调研
· PM 行 col1→col4 跨 col2/col3 双空长拉(.span-2!) + 3 个相邻 → + col7→col9 长拉 + col5 制定迭代↓分sprint
· 设计师 col2 → col3 相邻 + col3→col5 跨空 col4 长拉 + col5 内 ↓ + col5 → col6 相邻 + col6→col8 跨空 col7 长拉 + col8 → col9 相邻

为什么这次稳定:
· 每个箭头都"长"在它该在的 grid cell 里(::after 或独立 div),跟着 grid 走
· 不依赖跨容器坐标计算,不依赖 viewBox 缩放
· cell 移动,箭头自动跟着移动
· **永远不会飘、不会错位、不会被遮挡**

跨角色协作怎么表达?不在 L1 全景画箭头(避免飘),改在 L2 单角色页面用 task 卡内文字角标(如"接自研究员")表达。L1 干净清爽,只有同角色顺序。

Tooltip · hover 用户原声气泡(viewport-aware,不超界)

真实报告里 tooltip 由 _base.html 内置 JS 控制, 用 position:fixed + z-index:9999。 JS 检测 viewport 边缘,trigger 在不同位置时 tooltip 自动反向/偏移,**永远在 viewport 内可见**。下面三种情况示意:

case 1:正常
tip 在 trigger 上方,水平居中
hover 我(中间)
case 2:贴右边缘
tip 右边贴 viewport,不超出
hover 我(贴右边)
case 3:贴顶
hover 我(贴顶)
tip 翻到下方
已实现机制(_base.html 第 264-280 行 JS):
① left + width > viewport.width - margin → tooltip 右边贴 viewport 右边
② left < margin → tooltip 左边贴 margin
③ top < margin(上方空间不够) → 翻到 trigger 下方

为什么不会被组件容器裁切:tooltip 是 position:fixed,**脱离任何父容器的 overflow:hidden 限制**,直接挂在 body 上。

之前 gallery v1 看到超界的原因:gallery 模板没引入 _base.html 的 tooltip JS,fallback 走到 CSS-only `:hover::after` 伪元素,这个伪元素是被 `.gallery-render` 的 `overflow:hidden` 裁了 —— **真实 report.html 不会**。

Codex B 阶段做对的事:新 gallery 模板要把 _base.html 的 tooltip JS 完整引入,确保 hover 行为跟真实报告一致。
