READ ME FIRST · 首次使用者必读

访谈逐字稿 → 可点开的画像报告

上传访谈文件，模型和你协作完成一份交互式 HTML 用户画像 & 用户旅程 报告。支持企业用户（2B）、开发者（2D）、消费者（2C）三类研究。 与你对齐画像分类方式后，输出 1 个或多个用户画像和旅程。

可切换画像 · 字段 hover 看原话 输入 .docx / .txt / .xlsx 产出 report.html 支持 断点续跑

0快速开始

一句话就能开跑

在对话里@ 本 skill（或提到「用户画像」「访谈分析」等关键词），附上访谈文件或给出路径即可。模型会先问研究目标，再带你走完字段对齐 → 抽取 → 渲染。

@ skill + 附件
@user-persona 帮我从这些访谈生成画像报告。
（把 .docx / .txt / .xlsx 拖进对话，或写路径：C:\你的路径\访谈资料\）
  • 支持格式：.docx、.txt、.xlsx；不必事先整理成 JSON。
  • 头像可不提供：未放自定义图时，skill 会按画像名匹配内置 assets/default-avatars/。
  • 跑完后对外只发 最终交付件-*/ 文件夹里的 report.html（浏览器打开即可）。

1研究目标

先定目标，再选路线

skill 启动后先问这三件事，确认后再进入三种产出形态。字段、分群、页面侧重都从研究目标出发，不是模型从数据里猜。

①给谁看？
产品 / 设计 / 运营 / 管理层 / 跨团队
②回答什么？
例：哪类用户最在意 X · 各角色差异在哪 · 上手卡点是什么
③支持什么决策？
例：功能优先级 · 路线选择 · 汇报对齐 · 体验改版方向

2选路线

最多回答四个问题，指向三种形态

顺着问下来即可；也可以跳过决策树，直接看 §3 三种形态 + 样例，选最接近的一种。

Q1受访者是什么类型？
企业内员工 / 开发者→主题 2B / 2D 个人消费者 / 用户→主题 2C
Q2这批人是同一种角色吗？
是，同一种→形态 A 不是，多种角色→继续 Q3
Q3文件里角色已经分开了吗？
是，文件夹 / 文件名已分好→形态 B · 已分组 否，混在一起→继续 Q4
Q4你更想先定好几类人，还是用图找差异？
先一起定分类依据→形态 B · 未分组 两个维度画坐标→形态 C · 二维 三到五个维度横向对比→形态 C · 多维

3三种产出形态

先选报告长什么样，再定协作路径

每种形态附可点开样例。形态 B、C 内部还有两条路径（方式 2–5），差别在启动时要不要多聊几步，最终报告结构一致。

心智模型：1 张典型画像（形态 A）· 多个 tab 各一类人（形态 B）· 首页一张定位 / 对比图 + 各类画像（形态 C）。
A
单画像合并 2C / 2B / 2D 均适用
同一种角色 · 多份访谈合成 1 张典型画像

例：5 位产品经理的访谈 → 合成 1 份「典型产品经理」画像，字段 hover 可看原话。

产出1 个画像（主画像 + 细节 + 旅程三 tab）+ 原话证据
样例 ↗ 2B · 保障型运维工程师（主画像 + 细节 + 旅程） ↗ 2C · 内行场景派
你拍板：展示完整字段池（★/□）；是否补头像 / 场景截图；是否加旅程页（与模型确认即可）。
B
多画像 · 按类别分开 2C / 2B / 2D 均适用
多种角色 · 报告里多个 tab，每类一张画像（可选总体旅程）

例：运维、开发、产品各 2 人 → 3 个 tab，每类一张画像。最终报告一样，差别只在启动时文件有没有帮你分好组。

方式 2 多画像已分组

文件夹或文件名已按角色分开，如 运维/ 调度/。直接按组出画像，省事。

方式 3 多画像未分组

多种角色但文件混在一起。先和你确认「按什么分」（岗位 / 技能 / 场景…）和 tab 上的中文名。

产出N 个画像 tab（各含细节 + 单角色旅程）+ 可选总体 L1 旅程
样例 ↗ 2B · DevOps 五角色 + 总体旅程 + 单角色 L2
你拍板：方式 3 需确认分类依据与类别名；是否加旅程页（L1 整体 + 各角色 L2）、头像与场景截图。
C
多画像 · 用图区分 2C 常用 · 2B/2D 亦可
适合还没想好有几类人、想用可视化呈现差异

例：5 位音乐用户 或 5 位调度员 → 首页一张定位图或对比图，再进入各类画像 tab（可选旅程）。

方式 4 二维坐标区分

选 2 个关键区分点（如专业度 × 付费意愿）→ 四象限 → 每象限一类画像。

方式 5 多维分布区分

选 3–5 个区分点 → 一张分布图横向比较多类用户，再进各类画像。

产出图首页 + K 类画像 tab + 可选旅程
样例 ↗ 2C · 二维矩阵（方式 4） ↗ 2B · 电力调度员二维矩阵（方式 4） ↗ 2C · 多维分布（方式 5）
你拍板：区分点、档位名、类型名、受访者落点；是否加旅程页（每画像各 1 页 L2）。

5准备清单

第一次跑之前

访谈文件 必需

  • .docx / .txt / .xlsx，1 份到几十份均可
  • 已按角色分文件夹 → 倾向形态 B · 方式 2
  • 报告内受访者默认脱敏（如「张医生」「米同学」）

研究目标 必需

  • 想清楚 §1 三问；没想好模型会引导你聊清楚
  • 不必提前选形态，研究目标定完再走 §2 决策树

画像头像 可选

  • 自定义 · 画像头像素材/ · 文件名建议 画像中文名.png
  • 默认库 · skill 内置 assets/default-avatars/ · 不提供自定义图时按画像名自动匹配
  • 字段对齐时模型会主动问你；只有默认库也匹配不到时才用占位

界面截图 可选

  • 界面截图/ · 典型业务场景 + 可选旅程关注点
  • 放好后模型列出文件名，请你确认对应哪个场景

6产出结构

一次运行 = 一个项目目录

发给同事的只有 最终交付件-*/ 文件夹。

用户画像报告输出/
└── <项目名>-<日期时间>/
    ├── 过程稿/                       ← 中间产物，可续跑用
    │   ├── 00-research-goal.json    研究目标
    │   ├── 01-paradigm.json         形态 + 方式选择
    │   ├── 02-classification.json   分类 / 区分点（部分路径）
    │   ├── 03-field-alignment.json  字段对齐结果
    │   ├── 04-personas.json         合并后的画像数据
    │   ├── extracted/               每份访谈的抽取结果
    │   └── logs/                    运行日志
    │
    ├── 画像头像素材/                 ← 你放头像图
    ├── 界面截图/                     ← 你放截图
    │
    └── 最终交付件-2C-华为音乐HiRes-5用户-2维/
        ├── report.html              入口文件
        ├── _design-tokens.css
        ├── _components.css
        ├── assets/                  实际用到的头像 / 截图
        └── 交付件说明.md
对外只发「最终交付件-*」。相对路径，拷走就能打开。
「过程稿/」可续跑。删某个 JSON 就从那步重做。

7中断续跑

跑一半停了，或想重做某步

不用从头来

启动时扫 过程稿/00–04 的 JSON，从断点接着跑。

重做某步：删对应 JSON（如 03-field-alignment.json → 从字段对齐重来）。

全部重做：删整个项目目录。换模型只需复制 skill + 工作目录。
