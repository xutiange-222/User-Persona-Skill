# toC 分类依据 + 区分点推荐库

这份文档为 R3(toC 已知分类逻辑)、R4(toC 2 维坐标图)、R5(toC 多维分布图)提供**常见分类依据**和**区分点推荐库**。

toC 远比 toB 灵活,因此本文档提供更丰富的推荐 + 灵活组合机制。模型推荐时**必须基于研究目标**回扣。

---

## 第一部分:toC 常见分类依据(R3 用)

R3 场景下(用户心里有按什么分的想法,但需要 skill 帮他细化),常见分类依据:

### 1. 按人口统计学分类

- 年龄段 / 性别 / 收入 / 职业 / 教育 / 城市级别
- 适用于:营销定位、人群细分、广告投放

### 2. 按生活方式分类

- 都市精致 / 田园慢生活 / 数字游民 / 居家育儿
- 适用于:产品调性匹配、内容营销

### 3. 按消费观分类

- 实用派 / 颜值派 / 品牌派 / 性价比派
- 适用于:定价策略、产品功能优先级

### 4. 按对产品的态度/使用深度

- 重度用户 / 常规用户 / 轻度用户 / 流失用户
- 早期接受者 / 大众 / 跟随者 / 守旧者
- 适用于:用户运营策略、留存设计

### 5. 按使用动机/场景分类

- 工作场景 / 学习场景 / 娱乐场景 / 社交场景
- 适用于:场景化产品设计

### 6. 按生命周期阶段

- 单身 / 新婚 / 育儿 / 空巢 / 退休
- 适用于:婚育用品、家庭服务、保险等

## 第二部分:toC 区分点推荐库(R4/R5 用)

R4/R5 场景下,模型内部可以叫**价值变量**。但跟用户沟通时必须说**区分点**:它的意思是"用来把用户分成几类的判断标准"。模型从访谈数据 + 研究目标双重出发,推荐 2 个区分点(R4)或 3-5 个区分点(R5)。

### 区分点的三个筛选标准

来自 UXR 经典方法论(参考用户笔记软件画像案例):

1. **覆盖性**:所有受访者在这个维度上都有行为表现(不是部分人才有)
2. **差异性**:同一维度上,用户行为有明显差异(没有差异就分不出类)
3. **决策相关性**:这个维度对产品/业务决策有核心影响(对应研究目标)

模型推荐区分点时**必须按这 3 个标准筛**,且在话术中显式说明。

### 常见区分点候选库

按场景分类。模型推荐时,先看研究目标,再从对应场景库里选。

#### 笔记/工具类产品

- 记录习惯(简单记录 / 编辑批注 / 收集管理创作)
- 使用深度(基础功能 / 进阶功能 / 高阶功能)
- 用户期待(简单快捷 / 美观好用 / 功能强大)
- 输出诉求(自留 / 分享 / 协作)

#### 电商/消费类产品

- 价格敏感度(反复比价 / 看重性价比 / 认可即买)
- 决策模式(冲动 / 比较 / 长期研究)
- 品牌偏好(强品牌 / 性价比 / 国货 / 进口)
- 购物渠道偏好(直播 / 搜索 / 推荐 / 内容种草)

#### 社交/内容类产品

- 内容偏好(资讯 / 娱乐 / 知识 / 社交)
- 参与深度(只看 / 点赞 / 评论 / 创作)
- 社交活跃度(高活跃 / 中活跃 / 潜水)
- 信任来源(KOL / 朋友 / 算法推荐 / 平台官方)

#### 工具/SaaS 类产品

- 专业水平(小白 / 普通 / 高级 / 专家)
- 使用频次(每日 / 每周 / 每月 / 偶尔)
- 接受度(被动接受 / 主动探索)
- 经验(使用经验少 / 多)

#### 健康/运动类产品

- 运动频次(每日 / 每周几次 / 偶尔)
- 目标导向(健康 / 减脂 / 增肌 / 专业训练)
- 自律程度(计划稳定 / 间歇坚持 / 随缘参与)

#### 通用候选(适合多种场景)

- 年龄段 vs 收入(经典 2 维)
- 接受度 vs 经验(新技术类产品)
- 自主性 vs 协作性
- 严谨 vs 灵活
- 保守 vs 创新

## 区分点推荐话术

R4(2 个变量)模板:

```
基于你的研究目标(给 [audience] 看,回答 [research_question]),
我从访谈数据里看到几个区分度比较大的区分点。区分点就是用来把用户分成几类的判断标准。我推荐用这 2 个作为坐标轴:

【横轴】[区分点 1 名称]
- 含义:[一句话定义]
- 受访者表现差异:[例:5 位受访者中,2 位高,2 位中,1 位低]
- 为什么选它:[回扣研究目标]

【纵轴】[区分点 2 名称]
- 含义、差异、为什么选它(同上)

这 2 个变量交叉成 4 个象限,每个象限对应一类画像。

这个选择你认可吗?
- 认可,继续划档位
- 换一个区分点(我列出其他候选给你看)
- 我心里有别的区分点(告诉我,我可以基于数据判断它合不合适)
```

R5(3-5 个区分点)模板类似,但没有"横轴/纵轴"概念。面向用户要解释为:

```
如果用 3-5 个区分点,我会做一张多维分布图。
你可以把它理解成一张用户类型对照表:每一行是一个区分点,每一列是一类用户,用来横向看几类用户到底差在哪。
```

## 档位划分(R4/R5)

每个区分点需要划分**档位**(用户在这个区分点上的表现类型)。

档位数量:

- R4 通常每个区分点 2 档,产生 4 象限
- R5 通常每个区分点 2-3 档,产生丰富的差异

档位划分原则:

1. **互斥**:一位受访者在某个区分点上只能落一个档
2. **覆盖**:所有受访者都能被某个档容纳
3. **有意义的差异**:档位之间的差异要对决策有意义,不是技术性切分
4. **两端命名准确**:R4 坐标轴的两端不能只叫"高 / 低"。必须写成能直接理解的两极命名,例如"反复比价 / 认可即买"、"自主探索 / 依赖推荐"、"功能深用 / 轻量使用"。

模型推荐档位的话术:

```
对于区分点【记录习惯】,我建议分 3 档:

档位 1:简单记录,收藏备忘
  - 5 位受访者中,1 位属于这档
  - 典型表现:只用记事本功能,不分类不打标签

档位 2:编辑批注,知识管理
  - 5 位受访者中,2 位属于这档
  - 典型表现:会用标签、分类、批注,但很少导出输出

档位 3:收集管理并创作输出
  - 5 位受访者中,2 位属于这档
  - 典型表现:会用引用、链接、生成文档,把笔记作为创作素材

这个档位划分你认可吗?可以调整数量或重命名档位。
```

## 受访者档位映射(R4/R5)

每位受访者在每个区分点上落一个档。映射后**必须显性化展示给用户**(SKILL.md 职责表 + 离群处理原则):

```
我把 5 位受访者在【记录习惯】上的映射如下:

档位 1(简单记录):张三
档位 2(编辑批注):李四、王五
档位 3(收集创作):赵六、钱七

【价格敏感度】上的映射如下:
档位 1(高敏感):张三、赵六
档位 2(低敏感):李四、王五、钱七

这个映射你认可吗?
- 认可
- 调整某位受访者的档位
```

用户拍板后才能进入聚类阶段。

## toC 用户类型命名硬规则

用户类型名称是给产品、设计、用研快速理解用的,必须短、直观、能一眼读懂。

- 最多 5 个汉字,不含“型”“用户”“人群”等尾巴时优先
- 优先使用“内行深听派”“高配氛围派”“入门体验派”这类人群直觉名
- 不要把坐标轴逻辑直接拼成名称,例如不要写“价值待教育型价格权益用户”
- 如果分类逻辑很复杂,把复杂解释放进副标题、矩阵说明或画像描述,不要塞进名称
- tab 和矩阵象限只放短名称,长解释放 hover 或正文
- 命名前先读给产品经理听一遍:5 秒内能理解才通过
- 禁止把两个区分点直接拼接成抽象词,如“价值直觉型深度”“价值待教育型内容尝鲜者”
- 如果必须解释判断逻辑,放到副标题、矩阵说明或 hover,名称只承担识别作用

## 第三部分:聚类生成画像(R4/R5)

完成档位映射后,把档位组合相似的受访者聚成一类画像。

聚类逻辑:

- R4:每位受访者在 2 个变量上各落一档,共 2×2=4 种组合,每种组合 = 一类画像
- R5:每位受访者在 M 个变量上各落一档,理论上可能有 2^M ~ 3^M 种组合,但实际数据下通常聚成 3-5 类

模型聚类时:
1. 列出所有出现的组合
2. 把样本数为 0 的组合标"该象限/类别为空"
3. 给每类画像起一个名字(基于该类的典型特征)

聚类结果**显性化展示给用户确认**:

```
基于档位映射,我聚出 4 类画像:

【效率者】(2人:李四、王五)
  - 记录习惯:档位 2(编辑批注)
  - 使用深度:档位 3(高阶功能)
  - 用户期待:档位 1(简单快捷)

【创意者】(1人:赵六)
  - 记录习惯:档位 3(收集创作)
  - 使用深度:档位 2(进阶功能)
  - 用户期待:档位 2(美观好用)

...

这个分类你认可吗?
- 名称需要调整
- 调整某位受访者的归属
- 某类样本太少要不要合并
```

画像名字模型可以推荐,但**用户可以重命名**。
